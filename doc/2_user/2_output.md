### *ApexHDL Documentation*
# 2.2. Outputs

## 2.2.1. Folder structure

- At the start of execution, a folder is created at `<output_folder>/<circuit_name>/`.
- At the end of execution, the folder's structure is the following:
    - `vhdl/` = all **VHDL files** generated throughout the process,
    - `sim/` = all **simulation artefacts**,
    - `syn/` *(if hardware reporting was executed)* = all **hardware reporting artefacts**,
    - `impl/` *(if on-chip validation was executed)* = all **on-chip validation artefacts**.
    - `apex_report.rpt` = **final report**.

## 2.2.2. Generation

- This stage culminates with a **standalone VHDL module**.
- The VHDL code of the evaluator is written in `vhdl/<circuit_name>.vhd`.

## 2.2.3. Simulation

- Simulation starts by generating the **exhaustive VHDL testbench**, in `vhdl/tb_<circuit_name>.vhd`.
- The testbench is executed and writes the outputs in `sim/outputs_<circuit_name>.csv`.
- These outputs are parsed in order to produce **SVG visualizations** in `sim/`.

## 2.2.4. Hardware reporting

- Before performing synthesis, the evaluator is wrapped into a minimalist *top-level*, in `vhdl/top_<circuit_name>.vhd`.
- Standalone synthesis is executed, and **reports generated by the EDA tool** are put in `syn/*.rpt`.

## 2.2.5. On-chip validation

- Before performing synthesis, the evaluator is wrapped into an appropriate interface (depending on the target), the resulting *top-level* being written in `vhdl/stream_top_<circuit_name>.vhd`.
- Synthesis, place and route are executed. The whole is programmed on the FPGA, exhaustively stimulated, and results are retrieved in `impl/outputs_<circuit_name>.csv`.
- Various artefacts are placed in `impl/`, most notably the same **SVG visualizations** than during simulation.

## 2.2.6. Final report

- A **summary of all retrieved metrics** is written in `apex_report.rpt` in the `Results` section.
- This section is preceded by the `Parameters` section, which details the architectural parameters used during generation.
